# 算法随记

这里用于沉淀算法学习过程中的题目记录、解法思路、复杂度分析和容易忽略的边界条件。每篇随记建议固定记录：题目链接、核心思路、代码实现、复杂度和复盘点。

## 每日记录

每日记录适合保留最短路径：先写清楚为什么这样做，再补充失败过的思路。后续可以按日期或题号追加单独页面。

推荐题解结构：

- 题目：记录平台、编号和难度
- 思路：说明关键状态、数据结构或遍历顺序
- 代码：保留可复用的 Java 实现
- 复盘：写出这题下次如何更快识别

## 动态规划

动态规划题解重点记录状态定义、转移方程、初始化和遍历顺序。不要只贴代码，否则复习时很难还原为什么这样转移。

```java
class Solution {
    public int maxProfit(int[] prices) {
        int hold = -prices[0];
        int cash = 0;

        for (int i = 1; i < prices.length; i++) {
            cash = Math.max(cash, hold + prices[i]);
            hold = Math.max(hold, -prices[i]);
        }

        return cash;
    }
}
```

## 二叉树

二叉树题目优先判断使用前序、后序、层序还是递归返回值。复盘时可以重点写清楚递归函数的语义。

::: tip 记录建议
递归题解里先写“当前函数返回什么”，再写终止条件和左右子树如何合并。
:::

## 图论

图论题目建议按建图方式、遍历策略和去重条件拆开记录。BFS、DFS、拓扑排序和最短路最好分别沉淀模板。
