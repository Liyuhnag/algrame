VitePress Learning Notes Style Design ​

Goal ​

Optimize the existing VitePress site from the default starter template into a focused personal knowledge base for:

  • Algorithm problem solutions
  • Java backend interview questions and answers
  • Topic-based review notes

The site should feel like a calm study workspace: readable, organized, and suitable for long-term daily note-taking.

Site Positioning ​

Use the broader name 阿航的学习笔记 instead of only 阿航的算法笔记, because the site now stores both algorithm notes and interview preparation material.

Suggested site description:

算法题解、Java 后端面试与知识复盘

Visual Direction ​

Adopt the selected 专注刷题工作台 direction:

  • Light, quiet background
  • Green as the primary accent color
  • Restrained card surfaces with subtle borders
  • Dense but comfortable reading layout
  • Code blocks that feel polished and easy to scan
  • No marketing-style hero or decorative visual noise

The design should prioritize repeated use: finding notes, reading solutions, and reviewing interview answers.

Homepage Structure ​

The homepage should present two primary entry points:

  • 算法题解: daily practice notes, solution comparisons, complexity analysis, and topic tags
  • 面试题库: Java backend interview questions grouped by technical area

The hero copy should clearly explain that this site is used to accumulate problem-solving ideas, interview answers, and review notes.

The right-side hero module should be 笔记速览, not 今日复盘. It should preview both content types:

  • Algorithm note examples such as dynamic programming, binary tree, and monotonic stack
  • Interview note examples such as Java collections, JVM, and MySQL transactions
  • A small code or keyword preview to reinforce the technical identity

Navigation ​

Top navigation should use Chinese labels aligned with the site purpose:

  • 首页
  • 算法题解
  • 面试题库
  • 专题索引

Remove the default VitePress Examples navigation from the visible experience.

Sidebar Information Architecture ​

Use a simple grouped sidebar:

  • 算法题解
    • 每日刷题
    • 动态规划
    • 二叉树
    • 图论
  • 面试题库
    • Java 基础
    • JVM
    • MySQL
    • Redis
    • Spring
  • 复盘与专题
    • 项目追问
    • 场景设计

This implementation pass can update configuration and labels without creating a large empty directory tree. Placeholder links may point to the current example pages only if needed to avoid broken navigation, but visible copy should no longer look like the VitePress starter.

Styling Scope ​

Create a VitePress custom theme stylesheet and import it from the local theme entry. The style pass should cover:

  • CSS variables for brand color, background, borders, text, and code surfaces
  • Navbar and sidebar polish
  • Homepage hero layout and action buttons
  • Feature cards and entry cards
  • Markdown article typography
  • Headings, links, blockquotes, custom containers, and code blocks
  • Responsive behavior for mobile widths

The implementation should stay close to VitePress default theme APIs and avoid custom Vue components unless styling alone is insufficient.

Non-Goals ​

This pass will not:

  • Build a complete content taxonomy with many empty note files
  • Add search integrations beyond VitePress defaults
  • Add dynamic progress statistics
  • Add a custom content database or generated index

Those can be added later after real notes accumulate.

Verification ​

After implementation:

  • Run the VitePress build command
  • Start or use the local dev server
  • Inspect the homepage and at least one markdown article in the browser
  • Check that the page is readable on desktop and a narrow viewport
