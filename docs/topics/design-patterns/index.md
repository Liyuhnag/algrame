# 设计模式专栏

这个专栏用于把设计模式从“背概念”整理成“能解释、能落地、能结合项目回答”的内容。整体上拆成三部分：设计原则与思想、设计模式与范式、开源与项目实战。

## 学习目标

- 不只记住模式定义，更要理解它解决的核心问题
- 能区分相似模式的适用边界和取舍
- 能把设计模式和项目代码、开源框架实现联系起来

## 目录

- [设计原则与思想](/topics/design-patterns/principles)
- [设计模式与范式](/topics/design-patterns/patterns-and-paradigms)
- [开源与项目实战](/topics/design-patterns/open-source-and-project-practice)

## 建议写法

每个专题建议按下面顺序整理：

1. 先回答“它解决什么问题”
2. 再说明“核心思想和结构”
3. 补充“优缺点和适用场景”
4. 最后落到“项目代码或开源框架里怎么体现”
