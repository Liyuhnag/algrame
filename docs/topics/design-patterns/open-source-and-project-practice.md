# 开源与项目实战

这一部分主要把设计模式和真实代码联系起来。目标不是停留在概念层面，而是能在 Spring、MyBatis、JDK 集合、消息中间件客户端或者自己项目里找到对应的设计思想。

## 推荐整理方向

- Spring 中的工厂、模板方法、责任链
- MyBatis 中的代理模式
- JDK 集合或并发包中的设计思想
- 自己项目里的策略模式、责任链模式、观察者模式
- 模式滥用的反例

## 写作建议

- 尽量结合源码类名和调用链
- 项目实战优先回答“为什么当时这么设计”
- 不只写“用了什么模式”，更要写“带来了什么收益和代价”
